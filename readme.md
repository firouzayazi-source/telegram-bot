# ربات + پنل وب استوک لند

ربات تلگرام فروشگاه موبایل به همراه پنل مدیریت وب با گرافیک مدرن.
هر دو به یک دیتابیس وصل‌اند: هر تغییری از تلگرام یا وب، روی هر دو دیده می‌شود.

-----

## ساختار فایل‌ها

```
.
├── app.py              ← راه‌انداز اصلی (بات + وب با هم)
├── bot.py              ← ربات تلگرام
├── requirements.txt    ← کتابخانه‌های موردنیاز
├── Procfile            ← دستور اجرا برای Railway
├── railway.json        ← تنظیمات Railway
└── webpanel/
    ├── web.py          ← سرور پنل وب (Flask + API)
    ├── templates.py    ← قالب‌های HTML (صفحه ورود و پنل)
    └── uploads/        ← عکس‌های آپلودشده از وب
```

-----

## متغیرهای محیطی (Environment Variables)

این مقادیر را در Railway → Variables تنظیم کنید:

|نام           |توضیح                       |مثال               |
|--------------|----------------------------|-------------------|
|`BOT_TOKEN`   |توکن ربات از BotFather      |`123456:ABC-DEF...`|
|`ADMIN_ID`    |آیدی عددی ادمین             |`123456789`        |
|`PANEL_USER`  |نام کاربری ورود به پنل وب   |`admin`            |
|`PANEL_PASS`  |رمز عبور ورود به پنل وب     |`یک رمز قوی`       |
|`PANEL_SECRET`|کلید رمزنگاری نشست (اختیاری)|`یک رشته تصادفی`   |


> اگر `PANEL_USER` و `PANEL_PASS` تنظیم نشوند، پیش‌فرض `admin` / `stockland` است — حتماً عوضشان کنید.

-----

## راه‌اندازی روی Railway

1. کل پوشه را در یک ریپازیتوری GitHub بگذارید.
1. در Railway یک پروژه جدید از روی همان ریپو بسازید.
1. در بخش **Variables** متغیرهای بالا را وارد کنید.
1. Railway خودش `python app.py` را اجرا می‌کند (طبق Procfile).
1. در بخش **Settings → Networking** یک دامنه عمومی (Generate Domain) بسازید.
1. آدرس دامنه را باز کنید → صفحه ورود پنل ظاهر می‌شود.

ربات و پنل وب هر دو با یک سرویس اجرا می‌شوند؛ نیازی به سرویس دوم نیست.

-----

## نکته درباره عکس محصولات

وقتی از پنل وب عکس آپلود می‌کنید:

1. عکس در `webpanel/uploads/` ذخیره می‌شود (برای نمایش در وب).
1. همان عکس به تلگرام فرستاده می‌شود تا `file_id` بگیرد (برای نمایش در ربات).

این یعنی محصولی که از وب اضافه کنید، عکسش در ربات هم نمایش داده می‌شود و برعکس.

> **توجه:** فضای ذخیره‌سازی Railway موقت است (با هر deploy پاک می‌شود). برای نگه‌داری دائمی عکس‌ها و دیتابیس، یک **Volume** در Railway به مسیر پروژه وصل کنید.

-----

## اجرای محلی (تست)

```bash
pip install -r requirements.txt
export BOT_TOKEN="توکن"
export ADMIN_ID="آیدی"
export PANEL_USER="admin"
export PANEL_PASS="رمز"
python app.py
```

سپس مرورگر را روی `http://localhost:8080` باز کنید.

## امکانات پنل وب

- **داشبورد:** آمار کاربران، محصولات، درخواست‌ها
- **محصولات:** مدیریت درختی دسته → زیردسته → محصول، با آپلود عکس
- **درخواست‌ها:** مشاهده و پیگیری درخواست‌های خرید
- **کاربران:** جستجو، فیلتر، بلاک/رفع بلاک
- **مدیریت بخش‌ها:** ویرایش متن و دکمه‌های هر بخش منوی ربات
- **ساعت کاری:** تنظیم شیفت‌های هر روز هفته
- **تنظیمات:** کنترل نمایش گزینه‌های ربات

تمام این امکانات از داخل ربات تلگرام هم در دسترس‌اند.
